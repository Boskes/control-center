🐙 GitHub Setup

Version control, collaboration, and CI/CD — step by step

1

Create a GitHub account

Go to github.com and sign up. It's free. Use a professional username — this becomes your portfolio.

2

Install Git locally

Download git-scm.com, install with defaults. Then configure your identity in the terminal below.

3

Create a repository

On GitHub: New → name it mike-control-center → Private → Create. Copy the HTTPS URL.

4

Push your code

Run the commands below to link your local workspace to GitHub and push everything.

5

Use branches

Work on new features in a separate branch. Merge via Pull Request when ready. Keeps main stable.

6

GitHub Actions (CI/CD)

Automate testing and deployment. Every push triggers your pipeline. Start simple, expand later.

⚙️ First-time Git setup

Run this once after installing Git. Replace with your actual name and GitHub email.

git config --global user.name "Your Name" git config --global user.email "you@example.com" git config --global init.defaultBranch main

🚀 Push this workspace to GitHub

Run these commands from C:\Users\jurge\.openclaw\workspace-main

cd C:\Users\jurge\.openclaw\workspace-main # Initialize (already done — repo exists) git add . git commit -m "feat: initial Control Center v2 setup" # Link to GitHub (replace URL with yours) git remote add origin https://github.com/YOUR_USERNAME/mike-control-center.git # Push git branch -M main git push -u origin main

🌿 Daily workflow (after setup)

These 3 commands are 90% of daily Git usage:

# See what changed git status # Stage and commit git add . git commit -m "fix: beschrijf wat je deed" # Push to GitHub git push

🛡️ Recommended .gitignore

Already created in your workspace. Excludes node_modules/, logs, and secrets.

node_modules/ *.log .env .env.local dist/ data/*.json ← keep if you want to track data in git ← or exclude to keep data local only

⚡ GitHub Actions — auto-test on push

Create .github/workflows/ci.yml to run tests automatically on every push.

name: CI on: [push, pull_request] jobs: test: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: actions/setup-node@v4 with: node-version: '20' - run: npm ci - run: npm test
